# 缠论看图 · chanlun-kline

按缠中说禅《教你炒股票》博客原文实现的K线分析应用：包含处理、分型、笔、线段、走势中枢、走势类型、MACD背驰、三类买卖点，全部规则以原文课号为据（见 [docs/缠论算法规格.md](docs/缠论算法规格.md)）。

零依赖：原生 ES Modules + Canvas，无 node_modules。

## 运行

```bash
# 静态服务（带 no-cache 头，避免浏览器缓存旧模块）
python3 serve.py 8722          # 然后打开 http://localhost:8722
# Claude Code 内：preview_start chanlun-kline（.claude/launch.json 已配置）

# 单元测试（22 项，含81课官方图例判例与多种子不变量 fuzz）
node tests/run-tests.mjs
```

## 使用

- **数据**：默认「教学走势·完整信号链」——手工设计的确定性行情，完整呈现：下跌趋势两个中枢 → MACD背驰 → 一买 B1 → 二买 B2 → 中枢三卖 S3×2 → 上行中枢 → 三买 B3×2。另有三种合成随机行情（可换种子）与 CSV 导入（列：date,open,high,low,close，支持中文表头）。
- **真实行情（界面直取）**：顶栏「股票」输入框输 6 位代码（如 `688719`，自动判沪深；指数带前缀如 `sh000001`）→ 选周期（日线/60/30/15/5分钟）→「看图」或回车。数据经 serve.py 代理拉取（日线=腾讯前复权，分钟=新浪不复权），自动带出股票名并进入「数据」下拉留档——同一只股票拉日线+30分+5分即可来回切换下钻（27课区间套的人工流）。命令行备用：`python3 tools/fetch_kline.py sh688719 800 m30` 产出 CSV 后从「CSV可多选」导入（离线/批量场景）。不支持从K线截图识别——缠论划分对高低点数值硬敏感（81课：0.08点之差改变整个划分），截图恢复的 OHLC 误差远超此量级。
- **笔规则**：严格（62/77课，顶底分型间须有独立K线，默认）/ 宽松（79/80课，分型不共用即可）。原文两口径并存未裁决，故做成开关。
- **图层开关**：合并K线框 / 分型箭头（77课：顶用向下箭头、底用向上箭头，放大后显示）/ 笔（黄）/ 线段（蓝粗）/ 段中枢（原文口径）/ 笔中枢（民间扩展，对照用）/ 买卖点 / MACD背驰对比带（蓝=进入段a，橙=离开段c）。
- **交互**：滚轮缩放、拖动平移、双击复位、十字线读数；虚线=未完成（笔/段/中枢）。
- **级别递归**（35课）：「级别递归」开关——线段→L1中枢→L1走势类型→当作次级别单元逐级上推；L2+ 中枢框（紫/金/玫红）叠加在主图，顶部级别泳道每级一行（红=上涨、绿=下跌、灰=盘整，虚边=未完成），悬停K线显示其所在各级走势类型；底部统计如「递归 L1:4枢4型 L2:1枢1型」。真实短历史常只有 L1（级别是长出来的，不是缺陷）；合成数据选 3200 根易见 L2。
- **缠师点评**：图下方自动生成六条结构解读（格局/当下段/位置/信号/力度/纪律），全部由当前图的几何结构确定性推导、句句带课号，切换数据即重算；金句均经108课语料逐字核对（规格 §12）。
- **历史分析**：右上「历史」面板——「保存当前分析」把 K线快照+笔规则+结论摘要（笔/段/枢/信号）+备注 **直接落盘到项目 `history/` 文件夹**（serve.py 提供接口；一个快照一个 JSON 文件，随坚果云同步，Finder 里可直接管理）；「载入」用当前引擎对快照数据重算重现；「导入JSON」把外来快照文件收进 history/。列表里保留保存时的结论文字，代码升级后载入重算若有出入可对照。早期存在浏览器 localStorage 的记录会在启动时自动迁移成文件。注意：历史功能需用 `python3 serve.py` 启动（纯静态服务器无落盘能力，面板会提示）。

## 部署

三种运行形态，同一代码自动适配（前端探测 `/api/*` 是否存在）：

| 形态 | 在线行情 | 历史存储 | 用法 |
|---|---|---|---|
| 本地 `python3 serve.py` | ✅ 代理 | 落盘 `history/` | 开发与自用，功能最全 |
| **GitHub Pages**（公网，零后端） | ✅ JSONP 直连（`js/quotesrc.js`：腾讯 `_callback` / 新浪 `jsonp_v2`） | 访客自己的浏览器 | 仓库 Settings 已开启，推送即发布 |
| Cloudflare Pages（可选加固） | ✅（`functions/api/quote.js` 代理，不依赖 JSONP 接口形态） | 访客自己的浏览器 | 注册 Cloudflare → Workers & Pages → Import Git repository → 选本仓库 → 默认 Deploy |

行情获取顺序：前端先探 `api/quote`（本地/CF 代理），无代理自动切 JSONP 直连。JSONP 依赖行情方接口形态不变，若某天失效，Cloudflare 版是稳定后备。

**发布真源与同步**：真源 = 坚果云项目目录（本目录）；GitHub 仓库是发布镜像（git 仓库不入坚果云，避免同步污染）。改动后同步命令：

```bash
rsync -av --delete --exclude .git --exclude history/ --exclude '*.csv' --exclude .claude/ --exclude __pycache__/ \
  "/Users/mikonos/Nutstore Files/Zettelkasten/AI-Zettelkasten/00_收件箱_工作区/projects/chanlun-kline/" ~/dev/chanlun-kline/
cd ~/dev/chanlun-kline && git add -A && git commit -m "sync" && git push
```
